# cardsTD 基本設計書（概念・外部設計）

「トランプ王国タワーディフェンス」の**基本設計**。何を・なぜこう作るかという概念レベルの設計を残す。内部実装の詳細は `docs/詳細設計書.md` を参照。

## 1. システム概要

cardsTD は、経路を進軍してくる「カード兵」を、プレイヤーが配置・強化する「トランプ兵士のタワー」で食い止める、1人用のリアルタイム・タワーディフェンス Web ゲーム。ブラウザ（Canvas）だけで完結する。

```
タワーを配置  ─▶  ウェーブ開始  ─▶  敵を撃破しゴールド獲得  ─▶  強化して次のウェーブ
（カードを設置）    （敵が進軍）        （自動攻撃）              （J→Q→K）
```

## 2. 設計思想（なぜこの構成か）

| 判断 | 理由 |
|------|------|
| **静的 Web（サーバー無し）** | 個人開発で運用負荷ゼロ・無料公開・高速。個人情報を扱わないため DB 不要。GitHub Pages に載せられる。 |
| **Canvas + バニラ JS（フレームワーク無し）** | リアルタイム描画（60fps ループ）に素直。依存ゼロで軽量、`file://` でも動く。学習・改造もしやすい。 |
| **ES modules を使わない `<script>` 構成** | `index.html` を直接開くだけで動作させるため（module は `file://` で読めない）。`window.CTD` 名前空間で共有。 |
| **カード＝兵士のキャラクター表現** | トランプの世界観を、カード本体に細い手足と頭を付けた「兵士」で表現。味方も敵も同じ描画関数で一貫させ、素材ゼロを維持。 |
| **バランス定義を1ファイルに集約** | タワー・敵・ウェーブを `js/config.js` にまとめ、非エンジニアでも調整しやすくする。 |
| **エース・ハイを採用** | トランプの一般的な強さ順に合わせ、味方は J→Q→K、最強の敵ボスを A（エース）とする。 |
| **ゆるい三すくみ（ソフトカウンター）** | ♠=装甲/ボス・♣=群れ・♦=長射程で万能、と役割を分ける。ただし相性差は約1.4倍の**緩い**優位に留め、どのタワーも全ての敵に有効（＝ペナルティ無し）。相性を外しても詰まず、「正解手順の暗記」を強いてストレスにならないようにするため。 |

## 3. 機能一覧（外部から見た機能）

| ID | 機能 | 概要 |
|----|------|------|
| F1 | タワー配置 | カードを選び、空きマスへ設置。 |
| F2 | 強化・売却 | タワーをタップして J→Q→K に強化、または売却。 |
| F3 | ウェーブ進行 | 「ウェーブ開始」で敵が出現、経路を進軍。 |
| F4 | 戦闘 | タワーが射程内の敵を自動攻撃（単発／連射／範囲）。 |
| F5 | 経済・ライフ | 撃破でゴールド、ゴール到達でライフ減少。 |
| F6 | 勝敗 | ライフ0でゲームオーバー、全ウェーブ生存でクリア。 |
| F7 | ゲーム操作 | 速度切替（x1/x2）・一時停止。 |

## 4. 画面構成の概念

**単一画面**。上から「HUD（ライフ／ゴールド／ウェーブ／速度／一時停止）」「盤面（Canvas）」「カードトレイ（タワー3種）」「フッター（ウェーブ開始ボタン＋ヒント）」。盤面の上に、タワー情報パネルとタイトル/勝敗のオーバーレイが重なる。

```
┌──────────── cardsTD ────────────┐
│ ❤20  🪙130  🌊0/10      x1  ⏸ │  ← HUD
│ ┌────────── 盤面(Canvas) ─────┐ │
│ │  道に沿って敵が進軍          │ │
│ │  空きマスにタワー(カード兵)   │ │
│ │  [タワーパネル: 強化/売却]    │ │
│ └────────────────────────────┘ │
│ [♠剣兵 50][♦弓兵 40][♣砲兵 70] │  ← カードトレイ
│ [ウェーブ開始 ▶]  ヒント        │  ← フッター
└──────────────────────────────────┘
```

## 5. データの概念モデル

```
マップ/経路(Path) ──進む──▶ 敵(Enemy) ◀──攻撃── タワー(Tower)
  ウェイポイント列        属性:HP/速度/報酬       属性:攻撃力/射程/
  道マス・設置可能マス      種別:数札/盾/絵札/A     連射/範囲・ランク
        ▲                                          
        └──── ウェーブ(Wave) が敵の出現を定義 ────┘
```

- **マップ/経路**：ウェイポイントの列から道マスを決め、敵はその中心を辿る。道以外の空きマスにタワーを置ける。
- **タワー**：味方の主役。種別（スート）ごとに攻撃特性が異なり、ランク（J/Q/K）で強くなる。
- **敵**：ウェーブで出現し経路を進む。種別ごとに HP・速度・報酬が異なる。
- **ウェーブ**：どの敵を何体・どの間隔で出すかの定義（全10）。

## 6. ゲームループの概念

1. 準備フェーズ：タワーを配置・強化。「ウェーブ開始」で戦闘へ。
2. 戦闘フェーズ：敵が順次出現し進軍。タワーが自動攻撃。撃破でゴールド、ゴール到達でライフ減少。
3. ウェーブ完了：全滅かつ出現し切ったら準備フェーズへ（ボーナス付与）。最終ウェーブ完了でクリア。
4. いつでも：ライフ0でゲームオーバー。速度切替・一時停止が可能。

## 7. 操作の概念

- **タワー配置**：トレイのカードをタップ→盤面の空きマスをタップ。armed 中は連続配置できる。
- **強化/売却**：配置済みタワーをタップ→パネルで操作。
- マウス・タッチ両対応（Pointer Events で統一）。

## 8. 非機能方針

- **対応**：PC・Android の最新ブラウザ。レスポンシブ（アスペクト比維持でフィット）。
- **性能**：`requestAnimationFrame` の 60fps ループ、ローカル完結で即応答。高 DPI 対応。
- **プライバシー**：収集・送信なし。外部通信なし。

## 9. 制約・前提

- バックエンド無し／LLM 無し／ビルドツール前提にしない（素の HTML/CSS/JS）。
- ホスティングは GitHub Pages。将来は Capacitor でアプリ化可能。

## 10. 将来拡張（概念）

- ボスの強化・特殊カード効果、サウンド、ハイスコア保存。
