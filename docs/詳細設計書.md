# cardsTD 詳細設計書（内部設計）

`docs/基本設計書.md` を実装レベルに落とした**詳細設計**。データ構造・アルゴリズム・描画・入力・デプロイ・テストを定義する。値は `js/config.js` の実装に対応する。

- **文書バージョン**：1.0
- **作成日**：2026-07-12
- **上位文書**：`docs/基本設計書.md`

---

## 1. 技術方針

- 素の HTML/CSS/JS（フレームワーク・バンドラ不使用）。`<script>` は**非モジュール**で読み込む。
- 全モジュールが `window.CTD` 名前空間に関数・定義を載せて共有する（ビルド不要、`file://` でも動作）。
- 状態は `CTD.Game` インスタンスのみ（永続化なし）。描画は Canvas 2D。

## 2. ファイル構成

```
index.html            # 器（HUD / 盤面Canvas / タワーパネル / オーバーレイ / トレイ / フッター）
css/style.css         # レイアウト・テーマ（カードテーブル調）、light 基調
js/config.js          # CONFIG / WAYPOINTS / TOWER_TYPES / ENEMY_TYPES / WAVES（バランス集約）
js/render.js          # drawSoldier（カード兵士）ほか盤面・弾・エフェクト描画
js/game.js            # CTD.Game：状態・エンティティ・シミュレーション
js/ui.js              # CTD.UI：HUD・トレイ・強化パネル・オーバーレイの生成と更新
js/input.js           # CTD.initInput：Pointer Events による配置・選択
js/main.js            # 起動・Canvasサイズ調整・ゲームループ
.github/workflows/pages.yml   # GitHub Pages への自動デプロイ
```

読み込み順（`index.html`）：config → render → game → ui → input → main。

## 3. 盤面・座標系（`config.js`）

- グリッド：`COLS=11 × ROWS=15`、`CELL=40`。論理サイズ `W=440 × H=600`（縦持ち）。
- Canvas は論理サイズ×`devicePixelRatio` で確保し、CSS で領域にフィット（アスペクト比維持）。
- `START_GOLD=130`、`START_LIFE=20`、`SELL_REFUND=0.7`、`HP_WAVE_SCALE=0.10`。

### 経路（WAYPOINTS）
```js
[[1,-1],[1,2],[9,2],[9,6],[1,6],[1,10],[9,10],[9,15]]
```
- 連続ウェイポイント間（行または列を共有）を辿ってできるセルが「道」。先頭 `[1,-1]` は画面外の出現点、末尾 `[9,15]` は画面外のゴール。
- `buildPath()` が道セル集合（`roadSet`）と、各ウェイポイントの**ピクセル中心列**（`path`）を生成する。

## 4. データスキーマ（`config.js`）

### 4.1 TOWER_TYPES（味方＝トランプ兵士・3種）
```js
{ id, name, suit, color, buildCost, projSpeed,
  hit:'single'|'splash', desc,
  levels:[ { rank, damage, range, fireRate[, splash][, upgradeCost] } ] } // 0=J,1=Q,2=K
```

| id | suit | 建設 | 特性 | J(dmg/range/rate) | Q | K |
|----|------|------|------|------|------|------|
| spade | ♠ | 50 | 単発高威力 | 18 / 95 / 1.0 | 30 / 105 / 1.1（強化45） | 52 / 120 / 1.25（強化85） |
| diamond | ♦ | 40 | 連射・長射程 | 6 / 110 / 3.5 | 9 / 122 / 4.3（強化40） | 13 / 138 / 5.2（強化75） |
| club | ♣ | 70 | 範囲（splash） | 11 / 85 / 0.8, r46 | 17 / 92 / 0.9, r54（強化60） | 26 / 100 / 1.0, r62（強化105） |

### 4.2 ENEMY_TYPES（敵＝カード兵・4種）
```js
{ id, name, hp, speed, reward, color, kind:'pip'|'shield'|'face'[, rank] }
```

| id | 種別 | HP | 速度 | 報酬 | 備考 |
|----|------|----|------|------|------|
| number | 数札兵 | 34 | 74 | 6 | 速い・脆い。ランクは 2〜10 をランダム表示 |
| shield | 盾持ち兵 | 120 | 44 | 13 | 遅い・硬い。盾アイコン付き |
| face | 絵札の騎士 | 280 | 48 | 48 | 中ボス、ランク K |
| ace | エースの王 | 620 | 40 | 120 | 最強ボス、ランク **A** |

- 出現時の実 HP ＝ `base × (1 + waveIndex × HP_WAVE_SCALE)`（後半ほど硬くなる）。

### 4.3 WAVES（全10）
```js
[ [{type,count,interval}, ...], ... ]  // 各ウェーブは1つ以上の「敵グループ」
```
- 例：W3 = 数札6(0.6s)＋盾2(1.2s)、W9 = 絵札1＋数札12、W10 = 盾6＋絵札2＋**エース1**。
- グループは順番に消化（グループ間に約 0.6s の間隔）。

## 5. ゲームループ（`main.js`）

- `requestAnimationFrame(frame)`。`dt = min(0.05,(now-last)/1000)`。
- `game.speed`（1 or 2）の回数だけ `game.update(dt)` を実行 → 2倍速を実現。
- `game.paused` 中は更新を止める。毎フレーム `CTD.drawScene(ctx,game)` と `CTD.UI.refresh()`。

## 6. シミュレーション（`game.js`）

### 6.1 状態（フェーズ）
`title` → `build`（準備）⇄ `wave`（戦闘） → `over` / `win`。`start()` で `reset()`＋`build`。

### 6.2 敵の移動
- `path[wp]` を目標に、`speed×dt` だけ前進。到達したら次のウェイポイントへ。累積移動距離 `traveled` を保持（ターゲット選択に使用）。
- 末尾（ゴール）到達で除去し `life--`。0 で `over`。
- `phase` を進めて描画側の**行進アニメ**（脚のスイング）に使う。

### 6.3 タワーの攻撃
- `cooldown` を減算し、0 以下で射程内の**最も進んだ敵**（`traveled` 最大）を狙って発射。`cooldown = 1/fireRate`。
- 弾（projectile）は対象へ**ホーミング**。命中時：`hit==='splash'` なら着弾点の半径内へ範囲ダメージ＋エフェクト、それ以外は対象のみ。
- ダメージで HP0 の敵は除去し `gold += reward`。

### 6.4 ウェーブ制御
- `startWave()`：ウェーブ定義を、時刻付きの出現キュー（`spawnQueue`）に展開。
- `update()`：`spawnClock` を進め、時刻に達した敵を `spawnEnemy()`。キューが空かつ敵0で、最終ウェーブなら `win`、それ以外は `build`＋ボーナス `+25`。

### 6.5 配置・強化・売却
- `canBuildAt`：グリッド内・非道・未占有。`placeTower`：ゴールド支払い＋`occupied` 登録。
- `upgradeTower`：次ランクの `upgradeCost` を支払い `level++`。`sellTower`：投資額×`SELL_REFUND` を払い戻し。

## 7. 描画（`render.js`）

### 7.1 drawSoldier（カード兵士）
カード本体（角丸白＋スート＋ランク）に、**細い手足**と**小さな頭**を付けて「兵士」を表現。味方・敵で共用。
- 脚：本体下から2本＋足先。`marching` 時は `phase` で交互にスイング。
- 腕：本体上部左右から。タワーは右手に**槍**（柄＋三角の穂先）を構える。
- 頭：本体上に肌色の円。
- 本体：`roundRect` の白カード、枠色＝スート色、中央にスート記号、左上にランク。

### 7.2 盤面・その他
- `drawBoard`：フェルト背景＋薄いグリッド＋道セル（角丸）。
- `drawPlacement`：配置中は設置可能マスを薄く強調、ホバーマスを緑（可）/赤（不可）で表示し、射程円をプレビュー。
- `drawEnemies`：兵士＋HP バー。盾兵は盾アイコン。`drawProjectiles`：弾＋範囲攻撃のリングエフェクト。
- 選択中タワーは射程円を表示。

## 8. UI（`ui.js` / `index.html`）

| 部品 | 実装 |
|------|------|
| HUD | ライフ／ゴールド／ウェーブ／速度ボタン／一時停止ボタン。毎フレーム更新。 |
| カードトレイ | `TOWER_ORDER` から生成。選択中は枠強調、ゴールド不足は淡色（`poor`）。 |
| タワーパネル | 選択タワーの情報＋強化（コスト・可否）／売却／閉じる。 |
| ウェーブ開始 | `canStartWave()` で活性制御。状態に応じたヒント表示。 |
| オーバーレイ | タイトル／ゲームオーバー／クリア。**フェーズ変化時のみ**再構築（ボタンが毎フレーム再生成されない）。 |

## 9. 入力（`input.js`）

- Canvas に Pointer Events を付与（マウス／タッチ共通）。
- 画面座標→論理座標：`getBoundingClientRect()` で `x=(clientX-left)×W/幅`（縦も同様）。
- `pointerdown`：配置中で空きマス→設置、既設タワー→選択して armed 解除。非配置時→そのマスのタワーを選択。
- `pointermove`：ホバー座標を更新（射程プレビュー用）。`contextmenu`：配置キャンセル。
- `touch-action:none` によりタッチのスクロール誤作動を防止。

## 10. デプロイ（`.github/workflows/pages.yml`）

- トリガ：`main` への push（＋手動 `workflow_dispatch`）。
- ジョブ：`actions/configure-pages` →`actions/upload-pages-artifact`（`path: .` ＝リポジトリ直下）→`actions/deploy-pages`。`permissions: pages:write, id-token:write`。
- **一度だけ手動**：Settings → Pages → Source を「GitHub Actions」に設定。
- URL（例）：`https://an-dot-su4.github.io/cardsTD/`。

## 11. テスト・検証

### 11.1 手動確認
- `index.html` を直接開く、または `npx http-server .` / `python3 -m http.server` で配信。

### 11.2 E2E（Chromium/Playwright, `executablePath: /opt/pw-browsers/chromium-1194/chrome-linux/chrome`）
実装時に以下を確認済み：
- タイトル→開始→タワー配置でゴールド減少（道・不足時は不可）。
- ウェーブ開始で敵が出現・進軍し、タワーが撃破してゴールド増加、ウェーブがクリアされる。
- タワー選択→強化（J→Q でコスト減算・レベル上昇）→売却（払い戻し）。
- コンソールエラーが出ないこと。スマホ幅（375px 相当）でのレイアウト。

## 12. 既知の非対応・注意

- セーブ・サウンド・複数マップ・ネットワーク対戦は未対応（スコープ外）。
- 実 APK ビルドは Capacitor でローカル/クラウド実施（README 参照）。
- Pages 公開にはリポジトリの一度きりの手動設定が必要。
