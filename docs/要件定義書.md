# cardsTD 要件定義書

トランプ（cards）をモチーフにした、1人用のリアルタイム・タワーディフェンス Web ゲーム。攻めてくる「カード兵」を、プレイヤーが配置する「トランプ兵士のタワー」で食い止める。

- **文書バージョン**：1.0
- **作成日**：2026-07-12
- **対象リポジトリ**：`an-dot-su4/cardsTD`

---

## 1. 背景と目的

### 背景
ユーザーが「Android で動くゲームを作りたい」という要望を出発点に相談を行い、ジャンルは **1人用のリアルタイム・タワーディフェンス**、モチーフは **トランプ**（英語で cards、リポジトリ名の由来）に決定した。まず Web ゲームとして完成させ、後に Capacitor でアプリ（APK）化できる形を目指す。

### 目的
タッチ操作と相性が良く、段階的に拡張しやすいタワーディフェンスを、外部素材ゼロ・オフライン動作の **HTML5 Canvas + バニラ JavaScript** で実装する。トランプの世界観を、カードに手足と頭を付けた「兵士」キャラクターとして表現する。

### ゴール（このプロダクトが提供する価値）
- スマホ（Android）でも PC でも、ブラウザを開くだけで遊べる。
- トランプ兵士を配置・強化して敵の侵攻を防ぐ、手応えのある防衛体験。
- サーバー・課金・外部通信なしで完結。後からアプリ化もできる。

## 2. 想定ユーザーと利用シーン

| 種別 | 想定 |
|------|------|
| 主なユーザー | 手軽に遊べるタワーディフェンスを探している人。ゲーム初心者〜中級者。 |
| 利用デバイス | Android スマートフォン、PC（ブラウザ）。 |
| 利用シーン | すきま時間に1プレイ。タッチでタワーを置き、ウェーブを防いでクリアを目指す。 |
| 前提知識 | 不要（ルールは画面内で完結）。 |

## 3. 機能要件

### FR-1 タワー配置
- 画面下の**カード（タワー）**をタップして選択し、盤面の空きマスをタップで配置する。
- タワーは3種：♠ スペード剣兵（単発・高威力）／♦ ダイヤ弓兵（連射・長射程）／♣ クラブ砲兵（範囲攻撃）。
- 道（敵の経路）・既設タワーのマスには置けない。所持ゴールドが足りない場合は置けない。

### FR-2 タワーの強化・売却
- 配置済みタワーをタップすると情報パネルが開き、**強化**（ランク **J → Q → K** に昇格、攻撃力・射程などが上昇）または**売却**（投資額の一定割合を払い戻し）ができる。

### FR-3 敵とウェーブ
- 敵は道に沿ってゴールへ進軍する。敵は4種：数札兵（速い・脆い）／盾持ち兵（遅い・硬い）／絵札の騎士（中ボス、ランク J/Q/K）／**エース(A) の王**（最強ボス）。
- 全 **10 ウェーブ**。各ウェーブは「ウェーブ開始」で出現を開始する。後半ほど数・硬さが増し、終盤に絵札・エースが登場する。

### FR-4 経済（ゴールド）とライフ
- 敵を倒すと**ゴールド**を得て、タワー配置・強化に使う。ウェーブ生存で少額ボーナス。
- 敵がゴールに到達すると**ライフ**が1減る。ライフが 0 でゲームオーバー。

### FR-5 勝敗
- ライフが 0 になると**ゲームオーバー**画面を表示。
- 全ウェーブを耐え抜くと**クリア**画面を表示。どちらからも「もう一度」で再挑戦できる。

### FR-6 ゲーム操作
- **速度切替（x1 / x2）** と **一時停止／再開** ができる。

### FR-7 入力（マウス／タッチ両対応）
- Pointer Events により、マウスとタッチの両方で同一操作ができる。

### FR-8 オフライン・素材ゼロ
- 画像等の外部素材を使わず、図形と文字（スート記号 ♠♥♦♣ 等）だけで描画する。外部通信を行わず、`index.html` を開くだけ（`file://`）でも動作する。

## 4. 非機能要件

| 分類 | 要件 |
|------|------|
| 対応環境 | 最新の Chrome/Edge/Safari（PC・Android）。 |
| レスポンシブ | スマホ幅〜PC幅で盤面がアスペクト比を保って収まり、横スクロールを起こさない。 |
| パフォーマンス | `requestAnimationFrame` による滑らかな描画。ローカル完結で即応答。 |
| 表示 | 高解像度（devicePixelRatio）対応で Canvas をくっきり描画。 |
| 入力 | タッチ操作でスクロール等の誤作動を起こさない（`touch-action: none`）。 |
| 言語 | 日本語のみ。 |
| プライバシー | 個人情報を収集・送信しない。 |

## 5. 制約・前提

- **バックエンド無し**（サーバー不要の静的サイト）。
- **LLM 不使用**・外部 API 依存なし。
- **ビルドツール不要**（素の HTML/CSS/JS。ES modules は使わず `<script>` 直読み）。
- **ホスティング**：GitHub Pages（リポジトリ直下を公開）。
- ゲームバランス（タワー・敵・ウェーブ）は `js/config.js` に集約し、容易に調整できる形にする。
- 将来 **Capacitor** でネイティブアプリ（APK）化できるよう、素材ゼロ・オフライン動作を維持する。

---

## 6. スコープ外（今回作らないもの）

- 複数マップ、ボスの多彩化、特殊カード効果などの拡張要素。
- ハイスコア・進捗のセーブ、実績、サウンド。
- ネットワーク対戦・ランキング。
- Play ストア配信（実 APK ビルドはローカル/クラウドで別途）。多言語対応。

---

## 7. 成功基準（受け入れ条件）

- [ ] カードを選び、空きマスをタップするとタワーが配置され、ゴールドが減る。
- [ ] 道・既設タワーのマスや、ゴールド不足時には配置できない。
- [ ] 配置済みタワーをタップして**強化（J→Q→K）**・**売却**ができ、ゴールドが正しく増減する。
- [ ] ウェーブを開始すると敵が経路に沿って進軍し、タワーが自動で攻撃して撃破するとゴールドが増える。
- [ ] 敵がゴールに達するとライフが減り、0 でゲームオーバー、全ウェーブ生存でクリアになる。
- [ ] 速度切替・一時停止が機能する。
- [ ] マウス・タッチの両方で操作でき、スマホ幅／PC幅で表示が崩れない。
- [ ] コンソールエラーなく動作する。

## 8. 用語

| 用語 | 説明 |
|------|------|
| タワーディフェンス（TD） | 経路を進む敵を、設置した防衛ユニットで食い止めるゲームジャンル。 |
| ウェーブ | まとまった敵の襲来単位。本作は全10。 |
| エース・ハイ | トランプの一般的な強さ順（2 < 3 < … < 10 < J < Q < K < A、A が最強）。 |
| スプラッシュ | 着弾点の周囲にも及ぶ範囲ダメージ。 |
| Capacitor | Web アプリをネイティブアプリ（APK 等）に変換する仕組み。 |
